## 変更内容
- 管理者ログイン状態の一般ダッシュボード（`Dashboard.jsx`）における右上ボタンの配色を差別化しました。
  - 「Alert History」ボタンはインディゴ（`bg-indigo-600`）のままとし、「Verification Mode」ボタンは白背景・インディゴ枠線（`bg-white border border-indigo-600 text-indigo-600`）のアウトラインスタイルに変更することで、どちらも青で分かりにくい問題を解決しました。
- 検証画面（`Verification.jsx`）のコンテナ幅とヘッダー構造を一般ダッシュボードと完璧に統一しました。
  - 画面最上部に広がり余白の差異を生んでいた白いヘッダーバー（`<header>`）を廃止。
  - 一般ダッシュボードと全く同一 of 共通コンテナ（`max-w-7xl mx-auto px-4 sm:px-6 lg:px-8`）のフラットな構造に変更し、その中に直接「タイトル」と「アクションボタン」を配置しました。これによりメニュー幅や左右の余白のズレを解消しました。

## 静的確認結果
- フロントエンドアセット（React/Inertia/Tailwind）について、`npm run build` を実行してエラーなく正常に Vite ビルド（本番ビルド）が通ることを検証しました。

## 検証手順
user様は以下の手順で動作をご確認いただけます。

1. 管理者でログインした状態で、一般ダッシュボード（`/`）を開き、右上の「Alert History」と「Verification Mode」のボタンが別々のデザイン（ベタ塗りインディゴ vs 白地アウトライン）で並び、識別しやすくなっていることを確認します。
2. 「Verification Mode」をクリックして検証画面へ遷移し、画面上部にあった白ヘッダーがなくなり、一般ダッシュボードと全く同じ左右余白・タイトル高さ・コンテナ幅でシームレスに表示されることを確認します。
