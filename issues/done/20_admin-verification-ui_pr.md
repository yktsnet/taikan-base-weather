## 変更内容
- 管理者専用の検証モード画面 `Verification.jsx` を、一般ダッシュボード（`bg-gray-100` や白カード）と統一されたライトテーマで本実装しました。
- 以下の 3 つの検証・監視用コントロールパネルを画面上に構築しました。
  - **負荷シミュレーター**: 投入件数（1,000 / 5,000 / 10,000件）をボタン選択して「負荷シミュレーション開始」を実行できるフォーム。POST API `/admin/api/load-test` を非同期で呼び出します。
  - **SQS キュー監視**: 2秒間隔で GET API `/admin/api/metrics` を自動ポーリングし、SQS（水位・気象）の未処理件数（Pending）と処理中件数（In-Flight）、さらにデッドレターキュー（DLQ）の滞留数をプログレスバーやアラートカードでリアルタイムに可視化するインジケーター。
  - **DB 書き込みパフォーマンス**: 直近 5 分間にデータベースへバルクインサートされた水位・気象データの合計登録件数をリアルタイム表示するカウンター。

## 静的確認結果
- フロントエンドアセット（React/Inertia/Tailwind/Axios）について、`npm run build` を実行してエラーなく正常に Vite ビルド（本番ビルド）が通ることを検証しました。

## 検証手順
user様は以下の手順で動作をご確認いただけます。

1. 管理者でログインした状態で、管理者検証画面（`/admin/verification`）にアクセスし、ダッシュボードと同じライトグレー背景・白カードの整ったデザインで表示されることを確認します。
2. 「負荷シミュレータ」パネルで件数（例: 1,000件）を選択し、「負荷シミュレーション開始」ボタンをクリックします。
3. 投入後、中央の「SQSキュー監視」パネルの未処理件数ゲージが瞬時に増え、バックグラウンドの BulkQueueWorker の消化に伴って数値が滑らかに減少し、数秒でゼロに戻ることを確認します。
4. 同時に、右側の「DB書き込みパフォーマンス」パネルの水位・気象データ追加件数カウンターが、処理された件数分（例: 水位1,000件）だけ増加することを確認します。
