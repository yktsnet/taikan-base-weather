# Jules指示書

Julesは本ファイルを最優先の指示として実行すること。

## 1. 動作フローとインプット
- 起動時に `issues/` ディレクトリ内から、現在対象となっている `status: open` のIssueファイルを特定し、その「対象」「内容」「確認」フィールドの指示に従うこと。
- 実装・修正を開始する前に、必ず `context/conventions.md` を読み、コーディング規約および設計判断を完全に遵守すること。

## 2. 実行制限・禁止事項
- **禁止**: `docker` コマンド全般の実行。
- **禁止**: `php artisan migrate` などのデータ変更を伴うコマンド、およびテスト実行（コンテナ操作・モック外実行はすべてuser様の責務）。
- **許可**: `php -l` によるシンタックスチェックなど、非破壊的な静的解析コマンドの実行。

## 3. 成果物およびPR作成
- タスク完了時、ブランチ作成操作はスキップし、現在のクラウドサンドボックス環境から直接PRを作成すること。
- PR作成時、`.git/pr_body.md` に以下の構成で内容を書き出し、`gh pr create --base main --title "{type}: {タイトル}" --body-file .git/pr_body.md` を実行すること。

### PRボディ構成 (`.git/pr_body.md`)
- ## 変更内容 (Issueの内容を展開)
- ## 静的確認結果 (コード検証結果、および変更ファイル一覧)
- ## 検証手順 (user様がローカルで動作確認するための手順を実装内容から判断して記載)
